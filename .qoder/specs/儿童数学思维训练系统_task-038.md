# 儿童数学思维训练系统 - 设计说明书

## 一、项目概述

### 1.1 目标用户
- 刚上完一年级、即将升入二年级的小朋友（6-7岁）
- 家长辅助使用

### 1.2 设计目标
- 激发数学兴趣，培养数学思维
- 重点培养"数量、关系、规律、空间、推理"的感觉
- 不是提前刷二年级计算题，而是让孩子感受数学是可以观察、操作、猜测、推理和解决问题的游戏
- 每次训练 15~20 分钟，宁愿孩子觉得"还想玩"

### 1.3 核心原则
- 游戏化：所有训练以小游戏、闯关、挑战形式呈现
- 鼓励试错：答错不惩罚，给出引导提示，允许重试
- 多元评价：不只看对错，关注尝试次数、思考路径
- 适龄设计：大字体、大按钮、色彩丰富、操作简单（点击/拖拽为主）
- 无需识字量要求：关键提示配语音或图标

## 二、技术选型

| 项目 | 选择 | 说明 |
|------|------|------|
| 前端框架 | React 18 + Vite | 现代、快速、生态丰富 |
| UI 样式 | Tailwind CSS + 自定义主题 | 快速开发、色彩丰富 |
| 动画 | Framer Motion | 流畅的过渡和反馈动画 |
| 拖拽交互 | @dnd-kit/core | 空间思维模块的拖拽操作 |
| 状态管理 | Zustand | 轻量、简单 |
| 数据持久化 | localStorage | 无需后端，数据存本地浏览器 |
| 音效 | Howler.js | 答对/答错/完成等音效反馈 |
| 部署 | 纯静态站点 | 可直接本地打开或部署到任意静态托管 |

## 四、六大训练模块详细设计

### 4.1 数感模块（占比 30%）

**训练目标：** 灵活拆数、合数，理解数的组成和大小关系

**游戏形式：**

| 游戏名称 | 玩法 | 训练点 |
|----------|------|--------|
| 凑十游戏 | 屏幕显示一个数（如7），孩子从候选中选出能凑成10的数 | 补数概念 |
| 数字拆分 | 显示一个数（如8），拖拽分成两组，看有几种分法 | 数的组成 |
| 快速点数 | 短暂显示一组图形（如点阵），孩子选出数量 | 数感直觉 |
| 数字找朋友 | "谁和7合起来是10？"从多个选项中点击正确答案 | 补数反应 |
| 比大小挑战 | 两个数/算式比较大小，选择 > < = | 数量比较 |
| 估一估 | 显示一堆物品，估计大约有多少个（给出范围选项） | 估算能力 |

**交互方式：** 点击选择 + 拖拽分组

---

### 4.2 数量关系模块（占比 25%）

**训练目标：** 理解"一共、还剩、多多少、少多少"等数量关系

**游戏形式：**

| 游戏名称 | 玩法 | 训练点 |
|----------|------|--------|
| 故事小剧场 | 动画展示情境（如分糖果），孩子选择正确算式 | 情境理解 |
| 摆一摆 | 用虚拟积木/圆片摆出题目描述的关系 | 具象化理解 |
| 谁多谁少 | 两排物品对比，拖拽标注"多""少""同样多" | 比较关系 |
| 条件小侦探 | 给出一段话，圈出有用的条件，划掉无关条件 | 信息筛选 |
| 一题多解 | 同一个问题，找出所有正确的解法 | 灵活思维 |

**交互方式：** 点击选择 + 拖拽摆放 + 圈画标注

---

### 4.3 规律发现模块（占比 10%）

**训练目标：** 观察、猜测、验证规律

**游戏形式：**

| 游戏名称 | 玩法 | 训练点 |
|----------|------|--------|
| 接龙填空 | 数字/图形序列，填入下一个 | 发现规律 |
| 找不同 | 一组中找出不同类的那个 | 分类观察 |
| 规律设计师 | 孩子自己用图形/颜色创造规律 | 创造表达 |
| 图形变换 | 观察图形排列规律，预测下一个 | 图形规律 |
| 对错判断 | 给出一个规律和答案，判断对不对 | 验证能力 |

**交互方式：** 点击选择 + 拖拽排列 + 自由创作

---

### 4.4 空间思维模块（占比 20%）

**训练目标：** 拼、摆、转、分、合的空间感知

**游戏形式：**

| 游戏名称 | 玩法 | 训练点 |
|----------|------|--------|
| 七巧板拼图 | 拖拽七巧板拼出目标图形 | 图形组合 |
| 图形旋转 | 看一个图形旋转后变成什么样，选择答案 | 旋转想象 |
| 数图形 | 大图形中包含几个小图形（如几个三角形） | 图形分解 |
| 走迷宫 | 根据方向指令或自由探索走出迷宫 | 方向感 |
| 对称画 | 给一半图形，选出/画出另一半 | 对称概念 |
| 积木计数 | 看立体图，数出用了几个小方块 | 空间想象 |

**交互方式：** 拖拽拼图 + 点击选择 + 路径绘制

---

### 4.5 逻辑推理模块（占比 10%）

**训练目标：** 根据条件逐步排除、推理

**游戏形式：**

| 游戏名称 | 玩法 | 训练点 |
|----------|------|--------|
| 谁是谁 | 根据2-3个条件，匹配人物和物品 | 排除推理 |
| 排序挑战 | 根据条件排出顺序 | 有序思考 |
| 真假话 | 判断哪句话一定正确 | 逻辑判断 |
| 小侦探 | 根据线索找出"嫌疑人" | 条件综合 |

**交互方式：** 点击选择 + 拖拽匹配 + 连线

---

### 4.6 问题解决模块（占比 5%）

**训练目标：** 面对开放问题，愿意尝试、寻找多种方法

**游戏形式：**

| 游戏名称 | 玩法 | 训练点 |
|----------|------|--------|
| 公平分配 | 把物品分给几个人，找出所有分法 | 枚举思维 |
| 凑钱游戏 | 用不同硬币凑出指定金额 | 组合思维 |
| 路线探索 | 从起点到终点找出所有/最短路线 | 路径规划 |
| 自由拼搭 | 用指定数量积木拼出不同形状 | 创造尝试 |

**交互方式：** 拖拽操作 + 自由探索 + 多答案提交

## 五、训练会话流程设计

每次训练 15~20 分钟，结构如下：

```
┌─────────────────────────────────────────────────┐
│  热身小游戏（3分钟）                              │
│  - 随机从数感/比大小中选一个轻松游戏              │
│  - 目的：进入状态，建立信心                       │
├─────────────────────────────────────────────────┤
│  核心思维任务（8分钟）                            │
│  - 根据当日模块，完成 3~5 道题                   │
│  - 难度递进：简单 → 中等 → 挑战                  │
│  - 答错给提示，可重试                            │
├─────────────────────────────────────────────────┤
│  动手闯关（5分钟）                                │
│  - 操作性强的游戏（拖拽、拼图、迷宫等）          │
│  - 趣味为主，降低挫败感                          │
├─────────────────────────────────────────────────┤
│  总结分享（2分钟）                                │
│  - 展示本次获得的星星/徽章                       │
│  - "今天你发现了什么？"引导表达                  │
│  - 预告下次内容，保持期待                        │
└─────────────────────────────────────────────────┘
```

## 六、激励与评价体系

### 6.1 即时反馈
- 答对：星星飞入 + 欢快音效 + 鼓励语（"太棒了！""你想到了！"）
- 答错：温和提示 + 引导（"再试试看""换个角度想想"），不出现"错误""失败"字样
- 完成：烟花/彩虹动画 + 徽章

### 6.2 成长记录
- 每日训练日历（打卡记录）
- 各模块星星收集进度
- 累计徽章墙
- 连续训练天数（小火焰图标）

### 6.3 难度自适应
- 连续答对 3 题 → 自动提升难度
- 连续答错 2 题 → 自动降低难度或给出更多提示
- 家长可在设置中手动调整难度级别（1-3级）

## 七、页面设计

### 7.1 首页
- 六大模块以彩色卡片/岛屿形式展示（类似游戏地图）
- 每个模块有可爱图标和名称
- 显示今日推荐训练（根据进度自动推荐薄弱模块）
- 顶部显示连续天数和总星星数

### 7.2 训练页
- 顶部：进度条 + 当前星星数
- 中间：题目/游戏区域（大字体、大图形）
- 底部：操作按钮（大尺寸、易点击）
- 右上角：语音朗读按钮（题目读出来）

### 7.3 成长记录页
- 日历视图显示训练天数
- 六维雷达图显示各模块发展情况
- 徽章收集墙

### 7.4 设置页
- 音效开关
- 语音朗读开关
- 难度级别调整
- 训练时长选择（15分钟/20分钟）
- 数据重置

## 八、题库设计原则

### 8.1 题目结构（JSON格式）
```json
{
  "id": "ns-001",
  "module": "number-sense",
  "game": "make-ten",
  "difficulty": 1,
  "question": "谁和 7 合起来是 10？",
  "type": "single-choice",
  "options": [2, 3, 4, 5],
  "answer": 3,
  "hint": "从7开始往后数，数到10需要几个？",
  "encouragement": "太棒了！7和3是好朋友！",
  "audio": true
}
```

### 8.2 题目数量规划
- 每个模块每个游戏：初始 15~20 题
- 三个难度级别各 5~7 题
- 总计约 200~250 题（初始版本）
- 部分题目支持随机生成（如凑十、比大小），可无限变化

### 8.3 随机生成策略
- 数感模块：数字范围、拆分方式随机生成
- 规律模块：起始数、公差、图形类型随机组合
- 逻辑模块：人物、物品、条件模板随机填充

## 九、视觉与交互规范

### 9.1 色彩方案
- 主色调：温暖明亮（橙、黄、天蓝、草绿）
- 每个模块一个主题色
- 背景：浅色/渐变，不刺眼

### 9.2 字体与尺寸
- 题目文字：24px 以上
- 按钮：最小 48x48px 点击区域
- 选项间距充足，防误触

### 9.3 动画原则
- 过渡动画：0.3s，柔和
- 反馈动画：星星飞入、烟花、弹跳
- 不使用闪烁、快速移动等可能引起不适的效果

### 9.4 音效设计
- 点击：轻柔"叮"
- 答对：欢快短旋律
- 完成关卡：庆祝音效
- 背景：可选轻柔背景音乐
- 所有音效可关闭
